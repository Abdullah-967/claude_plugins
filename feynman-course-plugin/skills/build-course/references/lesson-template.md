# Lesson Template Reference

This template defines the standard structure for a single lesson page in a Feynman-style course. Every lesson should follow this format to ensure consistency and pedagogical effectiveness.

---

## Template

```
# Lesson [N]: [Title]

## Overview
[2-3 sentence plain-English summary of what this lesson covers and why. No jargon unless it was defined in a previous lesson. The learner should immediately understand what they're about to learn and why it matters to them.]

## The Big Idea
[First-principles explanation of the core concept. Start from what the learner already knows and build up. Break the idea into its smallest atomic parts, then reassemble them into understanding. Do not skip logical steps. Each sentence should follow naturally from the previous one.]

## Think of It Like...
[Intuitive analogy that maps the abstract concept to something familiar. The analogy should be concrete, relatable, and structurally accurate - not just superficially similar. Call out where the analogy breaks down if relevant.]

## In Practice
[Concrete, real-world example showing the concept in action. Walk through the example step by step. The example should be specific enough to be vivid but general enough that most learners can relate to it.]

## Why This Matters
[Connect this lesson back to the learner's stated goal. Answer: "So what? Why should I care about this?" Show how this concept is a building block for what comes next or how it changes the way the learner can think or act.]

## Key Terms
[Bulleted list of new terms introduced in this lesson with plain definitions. Each definition should be understandable without any other technical knowledge. If a term builds on a previous lesson's term, reference that lesson.]

- **[Term 1]**: [Plain-English definition]
- **[Term 2]**: [Plain-English definition]
- **[Term 3]**: [Plain-English definition]

## Check Your Understanding
[One question to verify the learner grasped the core idea. Should be conceptual, not trivia. The question should require the learner to think, not just recall. Good formats: "Why does...", "What would happen if...", "How would you explain... to a friend?"]
```

---

## Usage Notes

- **Lesson length**: Aim for a reading time of 5-10 minutes per lesson. Shorter is better if the concept allows it.
- **Tone**: Conversational, direct, no filler. Write as if explaining to a smart friend who just hasn't encountered this topic.
- **Jargon rule**: Never use a term before defining it. If a term was defined in a previous lesson, briefly remind the learner.
- **Analogy quality**: The "Think of It Like..." section is the heart of Feynman-style teaching. Spend extra effort making this analogy precise and memorable.
- **Check Your Understanding**: This is not a quiz. It is a self-check. The learner should be able to answer it if they understood the lesson. Provide the answer on the next lesson page or in the FAQ.
