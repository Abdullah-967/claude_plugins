# Course Template Reference

This template defines the standard structure for the course home page. It serves as the learner's entry point and navigation hub.

---

## Template

```
# [Topic]: A First-Principles Course

**Level**: [Beginner/Novice/Intermediate/Advanced/Expert]
**Lessons**: [N] lessons
**Estimated time**: [X] minutes

## What You'll Learn
[Bulleted list of learning objectives. Each objective should start with an action verb: "Understand...", "Explain...", "Apply...", "Distinguish...", "Build...". Aim for 3-6 objectives that are specific and measurable.]

- [ ] [Objective 1]
- [ ] [Objective 2]
- [ ] [Objective 3]

## Prerequisites
[What you should know before starting, or "None" for beginners. Be specific: "Basic familiarity with X" is better than "Some experience." If the learner is unsure, point them to a quick self-test or a simpler course.]

## Course Map
1. [Lesson 1 Title] - [one-line description]
2. [Lesson 2 Title] - [one-line description]
3. [Lesson 3 Title] - [one-line description]
...

## Supporting Materials
- Glossary
- Cheat Sheet
- FAQ & Common Misconceptions
- Practice Exercises

## About This Course
This course was generated using Feynman-style first-principles teaching. Each lesson builds on the previous one, using analogies and real-world examples to make complex ideas feel simple.
```

---

## Usage Notes

- **Topic naming**: The course title should name the topic plainly. Avoid clever names or subtitles. "Machine Learning" not "The Thinking Machine: A Journey Into ML."
- **Level selection**: Use the learner profile from the diagnose-and-map phase. The level determines lesson count, depth, visual complexity, and vocabulary.
- **Time estimation**: Estimate 5-10 minutes per lesson for reading, plus 2-3 minutes per visual, plus 5 minutes for the mastery check. Round to the nearest 5 minutes.
- **Learning objectives**: These come directly from the course plan generated by diagnose-and-map. Each objective should map to at least one lesson.
- **Course Map**: Each lesson title should be descriptive enough that the learner can tell what it covers at a glance. The one-line description adds context but should not exceed 15 words.
- **Supporting Materials**: These are generated by the build-course skill alongside the lessons. The glossary collects all Key Terms from every lesson. The cheat sheet is a one-page summary. The FAQ addresses common misconceptions identified during diagnosis.
